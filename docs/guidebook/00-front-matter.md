# The Free85 Guidebook

The Free85 Guidebook is the complete reference for Free85, the open-source
scientific graphing calculator firmware for the TI-85-compatible machine layer
in this repository. It is the companion to the Free85 Getting Started Manual:
the manual walks you through the front door, and this book covers each subject
in depth, one chapter at a time.

## Clean-room and licensing notice

Free85 is clean-room software, written from scratch for this project. Its
source code, generated artifacts, and public tests contain nothing taken or
derived from Texas Instruments ROM bytes, disassembly, fonts, artwork, binary
tables, or proprietary fixtures; mathematical expectations come from
independent reference results. The TI-85 is mentioned only to describe the
hardware profile the firmware runs on, and the project is not affiliated with
or endorsed by Texas Instruments. Because the internals are original, Free85
promises no binary, program, file, token, ROM-call, or internal-data
compatibility with TI software. What it does promise is that every physical
key, printed shifted function, alpha entry, and reachable built-in menu leads
to a meaningful Free85 feature.

Free85 is open source under the MIT License. See the `LICENSE` file for the
license text and `NOTICE.md` for the project notices.

## How to read this book

- **Keys** appear in brackets using their keycap labels: [ENTER], [2nd],
  [ALPHA], [F1] through [F5], [MORE], [x²], and the cursor keys
  [▲] [▼] [◀] [▶]. A sequence such as [2nd] [ENTER] means press and release
  [2nd], then press [ENTER].
- **On-screen text and typed expressions** appear in code spans: the status
  line shows `RAD AUTO`, and typing [2] [+] [3] puts `2+3` on the entry line.
- **Screenshots** are exact captures of the emulated 128 by 64 pixel LCD.
  Every image in this book is generated by booting a fresh machine, pressing
  the listed keys, and photographing the result, so what you see is what the
  calculator actually shows.
- **Planned-work callouts** flag features that are reachable in the chapters'
  subject areas but not implemented yet. They always look like this:

  > ⚠ **Planned:** what is missing (Free85 2.0, work package id).

- **Hardware callouts** flag features that work fully in the emulator but
  whose physical-hardware behaviour is validated and reported separately:

  > 🔌 **Hardware:** what works in the emulator — physical-cable validation
  > is reported separately.
